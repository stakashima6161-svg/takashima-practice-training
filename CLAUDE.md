# CLAUDE.md

## プロジェクト名

髙島事務所 実務トレーニング

GitHubリポジトリ名候補：`takashima-practice-training`

## 目的

髙島聖也税理士事務所の職員が、税務・不動産経営・相続・リスマネ・WSC報告を実務判断で学ぶためのGitHub Pagesサイトを作成する。

単なる知識確認ではなく、次の力を育成する。

- 資料から確認事項を見つける力
- お客様に安全に質問する力
- 自分で判断しすぎず、所長・主任へ上げる力
- WSCで簡潔に報告する力
- 不動産経営、相続、リスマネ、税務をつなげて考える力

## Ver.1 の要件

- 初級編100問を搭載
- 分野別に問題を絞り込める
- 出題数を10問、20問、30問、全問から選べる
- ランダム出題ができる
- 自動採点ができる
- 正答率、合格判定を表示する
- 分野別得点を表示する
- 間違えた問題に解説を表示する
- WSC報告用に、受講結果をコピーできる欄を表示する
- GitHub Pagesだけで動く
- 外部ライブラリは使わない
- 関与先情報、個人情報、金額、証券番号などを入力させない

## 分野

- 巡回監査
- 法人税
- 消費税
- 所得税
- 相続
- 不動産経営
- リスマネ・経営リスク
- WSC報告・お客様対応

## レベル設計

### 初級

対象：新人〜入所1、2年目
目的：正しく確認し、危ない判断をしない。

### 中級

対象：入所3〜5年目
目的：違和感や論点に気づき、所長に上げる。

### 上級

対象：主任・幹部候補
目的：複数論点を整理し、提案前の判断材料を作る。

## 問題データ形式

`data/questions.json` に以下の形式で保存する。

```json
{
  "id": "Q001",
  "level": "初級",
  "category": "巡回監査",
  "skill": "事前確認",
  "question": "問題文",
  "choices": ["A", "B", "C", "D"],
  "answer": 0,
  "explanation": "解説文"
}
```

## 文体ルール

- 職員教育向けに、難しすぎる専門用語を避ける
- ただし、税務・相続・リスマネの実務上の正確性は大切にする
- 「断定せず確認する」「所長へ上げる」「WSCに記録する」という実務姿勢を重視する
- リスマネ分野では、保険募集に見える表現を若手職員にさせない

## リスマネ分野の注意

リスマネの問題は、保険商品を売る訓練ではない。
目的は、経営リスクに気づき、所内のリスマネ担当・届出済み募集人・所長へつなぐこと。

若手職員が避ける表現：

- この保険に入りましょう
- この補償を付けた方がいいです
- 今の補償では足りません
- この代理店に見てもらいましょう

若手職員が使える表現：

- 内容は所内で確認します
- 経営上、どのような支出や影響が出るか確認しておきましょう
- 必要に応じて、担当者から改めてご案内します
- WSCに確認事項として残します
